# Quick Notes

A personal repository for technical notes and software engineering concepts.

## Structure

- **principles/** - Software engineering principles (SOLID, Clean Code, etc.)
- **design-patterns/** - Design patterns organized by type
  - creational/ - Factory, Builder, Singleton patterns
  - structural/ - Adapter, Decorator, Facade patterns
  - behavioral/ - Observer, Strategy, Command patterns
- **architecture/** - Software architecture patterns and concepts
  - microservices/ - Resilience patterns, communication patterns
  - monolithic/ - Monolithic architecture patterns
  - event-driven/ - Event-driven architecture patterns
- **best-practices/** - Testing, security, performance guidelines
- **tools/** - Frameworks and libraries documentation

## Usage

Add your notes as markdown files in the appropriate folders. Use descriptive filenames and include:

- Brief description
- Code examples
- Use cases
- References

## Getting Started

1. Navigate to the relevant folder
2. Create a new `.md` file
3. Add your notes and examples
4. Commit and push your changes
